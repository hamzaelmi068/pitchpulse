# UI Review Checklist

- Verify the screen still feels cohesive with the existing PitchPulse visual direction.
- Verify hierarchy, contrast, spacing, and readability on desktop and mobile.
- Verify critical interactions remain clear without relying on hidden states or hover-only behavior.
- Verify map, stats, and bracket content do not compete for attention in the same view.
- Verify loading and empty states feel intentional rather than placeholder-like.
- Record any durable design direction changes in `../memory.md`.
