# Feature Launch Checklist

- Confirm the feature has a clear user-facing goal and fits the current milestone.
- Confirm any dependency on sports-data, AI, or news providers is explicitly documented.
- Verify loading, empty, and error states exist where user data may be delayed or unavailable.
- Verify mobile and desktop behavior for all touched UI flows.
- Verify changes align with the map-first PitchPulse product direction.
- Update `../memory.md` if the work settles a lasting decision or assumption.
- Check git status to ensure only intended files changed.
